Mandato Parlamentar Digital · conceito de interface

Um sistema de gabinete que parece um sistema — sem virar mais um dashboard.

O MPD é uma ferramenta de trabalho diário para a equipe parlamentar. Cada demanda é um caso aberto com partes, histórico, encaminhamentos a órgãos externos e uma devolutiva ao cidadão. A metáfora não é planilha — é um registro institucional, lido como documento. Esta é a interface comprometida; abaixo, o mapa de telas: as que já estão construídas e as próximas.

Entrar como Mariana → Ver lista de demandas → Abrir uma demanda → Triagem do inbox →
Princípios visuais que sustentam o sistema
01

Documento, não dashboard

Cada tela é um documento institucional: cabeçalho com filete navy, número monoespaçado, título em serif, marginalia para datas.

02

Sans no UI, serif no relato

IBM Plex Sans no chrome e nos rótulos. Source Serif 4 nos relatos, descrições, devolutivas. Mono nos números frios.

03

Sem cantos arredondados

Réguas finas, ângulos retos, hierarquia tipográfica. Status como mini-carimbo, não como pílula colorida.

04

Apartidário por construção

Papel cru, tinta navy, bordô para alerta, verde-musgo para concluído. Nenhuma cor evoca partido. O sistema serve a qualquer mandato.
